Олегово расширение #2

ШАГ 1.3 — ТИП ТЕЛЕСНОСТИ (КРИТИЧЕСКИ ВАЖНО)

Определи, в каком теле человек проживает эту жизнь.

Выбери один доминирующий тип телесности:

  • grounded_body → устойчивое, укоренённое, опорное тело
  • light_body → лёгкое, свободное, дышащее тело
  • strong_body → сильное, собранное, живое тело
  • calm_body → спокойное, мягкое, регулируемое тело
  • vital_body → энергичное, тёплое, включённое тело
  • elegant_body → пластичное, красивое, уверенное в движении тело
  • restored_body → восстановленное, свежее, ресурсное тело

Правило

Тело должно быть не просто «видимым».

Сцена должна передавать: - как это тело двигается - как оно чувствует пространство - как оно держит ритм - как через него считывается качество жизни

ANTI-PATTERN MODULE — ВАРИАТИВНОСТЬ СЦЕН (КРИТИЧЕСКИ ВАЖНО)

Задача: НЕ повторять визуальные паттерны.

Перед генерацией финальной сцены ОБЯЗАТЕЛЬНО проверь:

1. Проверка паттерна

Если сцена выглядит как: - человек в центре + другие вокруг - человек объясняет / говорит / ведёт - группа слушает - «красивая активность» без уникальной структуры - универсальная сцена, которую можно применить к любой сфере

→ это повтор шаблона → СЦЕНА НЕВАЛИДНА → пересобери

2. Разнообразие композиции

Каждая сцена должна отличаться по композиции. Выбери один из вариантов и не используй один и тот же паттерн по умолчанию: - человек в центре - человек сбоку - человек частично в кадре - человек вне фокуса - человек как часть среды - сцена без человека - POV (от первого лица)

Правило: Нельзя использовать «человек в центре» как дефолт.

3. Разнообразие действия

Сцена НЕ обязана содержать активное действие.

Допустимые режимы: - активное действие - мягкое взаимодействие - наблюдение - пауза - статичность - отсутствие действия

Правило: Если можно убрать действие и сцена станет сильнее — убери его.

4. Разнообразие роли

Человек НЕ должен всегда быть: - лидером - центром внимания - говорящим - управляющим

Допустимо: - равный среди равных - наблюдающий - присутствующий - включённый без доминирования - вообще отсутствующий

5. Разнообразие сцены

Сцены не должны повторять один и тот же формат:

Запрещён дефолт: - разговор стоя - мини-группа - «обсуждение» - «объяснение идеи»

Используй разные типы сцен: - движение - переход - момент до / после - наблюдение - поле - тишина - совместное присутствие - следы жизни - пространство без действия

6. Разнообразие камеры

Меняй язык съёмки: - wide shot - medium shot - close shot - POV - layered composition

Правило: Камера должна соответствовать смыслу сцены, а не привычному шаблону.

7. Разнообразие энергии

Сцены должны отличаться по внутреннему состоянию: - динамика - спокойствие - тишина - напряжение - лёгкость - глубина - поток - наблюдение

Нельзя, чтобы все сцены были «живые, активные, включённые» одинаково.

8. Уникальность сферы

Проверь: Можно ли эту сцену перенести в другую сферу без изменений?

Если да → сцена слабая → пересобери.

Каждая сцена должна быть: - уникальной для своей сферы - не пересекаться по логике с другими сферами - не повторять визуальный язык других сфер

9. Финальная проверка

Если сцена: - похожа на предыдущие - использует тот же тип композиции - использует тот же тип действия - снова делает человека центром без необходимости - выглядит как универсальный «красивый момент»

→ пересобери сцену

Главный принцип

Каждая сцена = новая визуальная логика.

Не вариация. Не повтор. Не перестановка.

А другая структура, другая энергия, другой способ присутствия.

2. Двумерная классификация — Механика × Тело

ШАГ 1.2 (Механика Состояния, 7 типов: body / space / rhythm / energy / recovery / embodied / mixed) × ШАГ 1.3 (Тип Тела, 7 типов: grounded / light / strong / calm / vital / elegant / restored) = 49 кросс-комбинаций.

Это сильнейшая защита от шаблонной генерации. Каждый юзер получает уникальную пару (например, space_regulation × calm_body = тихое утреннее пространство, мягкое спокойное тело; energy_activation × vital_body = беговая дорожка с пульсом, тёплое включённое тело).

Гипотеза (моя): при генерации AI должен явно вывести обе классификации в начале промта (не «осознавая внутри», а в тексте). Тогда сцена строится на чёткой паре. Без этого — снова свободная интерпретация.
