3.2 BUG-2: {onboarding_answers} не попадает в 9 из 10 sphere prompts

Что происходит:
В GP ID 3–10, 32 в ВХОДНЫЕ ДАННЫЕ написано:

Общие ответы пользователя:
{profile_summary}          ← ЭТО profile_summary, НЕ onboarding_answers!

Вместо:

Общие ответы пользователя:
{onboarding_answers}       ← должно быть это

Следствие: Sphere generation prompts получают {profile_summary} дважды (анализ профиля + онбординг-раздел). Реальные онбординг-ответы пользователя ({onboarding_answers}) в сцену НЕ попадают.

Исключение: GP ID=2 (self) — единственный промпт с правильной структурой: использует {profile_summary} ×1 и {onboarding_answers} ×1.

5. STYLE BEHAVIOR PER SPHERE

5.1 Clothing style selection mechanism

Стиль одежды выбирается через GP ID=1 (sphere_selector / three_angles_prompt): - Inputs: {sphere_title}, {sphere_short}, {sphere_definition}, {profile_summary}, {micro_adjustment_answers} - Output: ОДИН стиль из {user_clothing_styles} - Принцип: “как этот человек РЕАЛЬНО выглядел бы, если бы уже жил этой жизнью”

5.2 Style anti-patterns (глобальные)

Во всех сферах система блокирует: - startup founder cliché - crypto bro aesthetic - airport luxury traveler vibe - generic entrepreneur uniform - influencer energy - overdressed luxury - rich-person stock photo - techwear stereotype - fashion editorial vibe - AI-generated perfection

5.4 Recurring style collapses (production observations)

Из audit sessions 2026-05-14/15:

1. Горы + холодный свет + чашка (IDs 587-589):
Три угла sphere “Свобода” — все три получили одинаковую FAL-сцену (горный перевал, холодный свет, человек с чашкой). Причина: {scene_description} placeholder не резолвится в photo_session → три итерации генерировали одну и ту же scene.

2. Corporate aspiration collapse (realization sphere):
ID=4 prompt описывает “слава / реализация” — в слабых профилях коллапсирует в офисные сцены, ноутбуки, дорогие кресла.

3. Lifestyle collapse (freedom sphere):
ID=10 prompt (который генерирует “социальные сцены” вместо свободы) → generic social gathering, wine bar aesthetics.

9. CURRENT WEAK POINTS

9.1 BUG-1: Prompt Rotation Error — 5 из 10 сфер

Affected spheres: children, creativity, wisdom, freedom, environment

Impact: - Sphere “children” генерирует creation/novelty сцены (без детей, без наследия) - Sphere “creativity” генерирует wisdom/knowledge сцены (без творческого процесса) - Sphere “wisdom” генерирует freedom/travel сцены (без глубины/роста) - Sphere “freedom” генерирует social environment сцены (без личной свободы) - Sphere “environment” — дубликат freedom’s wrong prompt → identical outputs

Severity: Пользователи получают нерелевантные сцены для половины сфер.

9.2 BUG-2: {onboarding_answers} missing

Impact: 9 из 10 sphere prompts работают фактически без персонализации через онбординг-ответы. {profile_summary} — это синтез, но в нём может отсутствовать специфика отдельных ответов, нужных для visual scene building.

Исключение: GP ID=2 (self) правильно получает оба источника.

9.3 BUG-3: Duplicate prompts (freedom = environment)

Impact: Две сферы генерируют идентичные сцены — нулевая дифференциация.
